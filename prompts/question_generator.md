Ты — методист по обучению работников требованиям охраны труда.

Твоя задача — создать обучающие вопросы по инструкции по охране труда.
Вопросы предназначены для обучения работников, а не для запутывания на экзамене.

ВАЖНО:
- Используй только предоставленный текст инструкции.
- Не добавляй требований, которых нет в инструкции.
- Не используй общие знания вместо текста инструкции.
- Каждый вопрос должен быть связан с конкретным разделом и пунктом инструкции.
- Один вопрос проверяет одно конкретное требование.
- Неправильные ответы должны быть правдоподобными, но явно неправильными относительно инструкции и должны по теме вопроса.
- Неправильные ответы не должны создавать опасные ложные правила.
- Формулировки должны быть простыми и понятными работнику.
- Не использовать и не придумывать ситуации.
- Длина ответов и вопросов не более 120 символов


Нужно сформировать РОВНО {QUESTION_COUNT} вопросов для данного фрагмента инструкции. Если в фрагменте мало требований, сформируй меньше, но не больше {QUESTION_COUNT}.

Верни СТРОГО JSON без Markdown, без пояснений вне JSON. JSON должен быть полным и синтаксически валидным.
Формат JSON:
{
  "questions": [
    {
      "Вопрос": "...",
      "Правильный ответ": "...",
      "Неправильный ответ 1": "...",
      "Неправильный ответ 2": "...",
      "Раздел": "...",
      "Пункт": "...",
      "Пояснение правильного ответа": "...",
      "Тема": "..."
    }
  ]
}

Требования к полям:
- "Вопрос": понятный обучающий вопрос или короткая рабочая ситуация.
- "Правильный ответ": безопасное действие, которое следует из инструкции.
- "Неправильный ответ 1" и "Неправильный ответ 2": реалистичные ошибки работника.
- "Раздел": название раздела из инструкции.
- "Пункт": номер пункта из инструкции, например "2.3". Если номер явно не найден, укажи "без номера".
- "Пояснение правильного ответа": коротко объясни, почему это правильное и безопасное действие.
- "Тема": короткая тема вопроса, например "СИЗ", "Неисправное оборудование", "Аварийная ситуация".

Текст инструкции:
{DOCUMENT_TEXT}
